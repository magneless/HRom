## Приложение для проверки кандидатов при найме. Основной функционал: проверка резюме с помощью llm, анализ ответов на заранее заготовленные ответы с помощью llm.

### Backend должен иметь такой функционал:
* Регистрация пользователя.
* Вход пользователя.
* Восстановление пароля.
* Загрузка резюме.
* Получение всех моих загруженных резюме
* Анализ резюме с помощью llm с получением списка несоответствий и местами в тексте, где эти несоответствия находятся.
* Получение заранее заготовленных вопросов.
* Отправка ответов на эти вопросы.
* Анализ ответов с помощью llm.
* Получение, добавление, изменение собственных личных данных.
* Возможность покупки подписок различного уровня.
* Возможность ввода промокода в профиле. 

### В репозитории находятся:
* Архитекрута проекта.
* Директория со схемами DB.
* Директория с описанием end-point-ов микросервисов.

### Доп:
* Методы/функции, вывзваемые при получении сообщений из топиков message брокеров, в end-point-ах не прописаны. Поэтому у микросервиса LLM(отвечает за взаимодействие с LLM) нет end-point-ов, он "слушает" message брокер и работает при получении информации в топики: cv.uploaded/questions.uploaded, после обработки данных записывает результаты в топики cv.analyzed/questions.analyzed, которые в свою очередь "слушают" микросервисы соответсвующие.
* AAA микросервис при создании пользователя отправляет сообщение в топик user.created, который "слушает" User Profile микросервис. Обновление email идет через AAA, потому что email - это логин, идентификатор входа.
